# ``BSafeEngine``

A bit-exact Swift port of the DSP layer of the Phonicbloom BSAFE (Gecho /
MMXX T-APE) ESP32 firmware.

## Overview

`BSafeEngine` renders the eight built-in Viznut-style bytebeat songs, the
ring-buffer echo, the sine-waves wavetable voice, the 8-step sequencer and
the arpeggiator that ship on the BSAFE hardware. The DSP has no dependency
on AudioKit, UIKit or any Apple audio framework — it compiles on every
Apple platform and produces stereo `Float` samples ready for an
`AVAudioSourceNode` render block, an `AudioUnit` render callback, or any
other audio host.

Correctness is anchored by sample-for-sample comparison against WAVs
generated by the original C reference; the bytebeat song formulas use
Swift's overflow operators so signed-integer wraparound matches C exactly.

## Threading model

The library is designed for the classic audio pattern of *one UI thread
mutating parameters, one audio thread reading them*:

- ``SynthParameters`` is `@MainActor`-isolated; SwiftUI binds to it
  directly.
- Every UI mutation copies a value-type ``SynthParameters/Snapshot`` into a
  lock-guarded slot via `OSAllocatedUnfairLock`.
- The audio thread calls ``SynthParameters/snapshot()`` once per render
  buffer and reads from a local copy for the duration of that render — no
  locks on the per-sample path.

The rendering engines (``BytebeatEngine``, ``SineWavesEngine``,
``Arpeggiator``, ``StepSequencer``, ``SynthCore``) are single-owner on the
audio thread. They are marked `@unchecked Sendable` so they can be created
off-thread and handed to the audio callback.
